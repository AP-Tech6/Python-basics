why index starts with 0 in list and array in python?
In Python, as in many programming languages, indexing starts at 0 
because it allows for a more efficient way to calculate the memory address of each element in a list or array. 
When you access an element at index `i`, the memory address can be calculated as:
address = base_address + (i * size_of_element)
If indexing started at 1, the calculation would be: 
address = base_address + ((i - 1) * size_of_element)
This would require an additional subtraction operation, making it less efficient. 
Starting at 0 simplifies the calculation 
and is a convention that has been widely adopted in programming languages.

slicing
Slicing is a powerful feature in Python that allows you to extract a portion of a list, string, or any other sequence type. The syntax for slicing is:
sequence[start:stop:step]

l1=[21,32,13,4,85,62,47,77]
print(l1[2:5]) // it will print from index 2 to index 4
output: [13, 4, 85]
print(l1[-4:-1]) // it will print from index -4 to index -2
output: [85, 62, 47]
print(l1[-4:]) // it will print from index -4 to the end of the list
output: [85, 62, 47, 77]
print(l1[::-1]) // it will print the list in reverse order
output: [77, 47, 62, 85, 4, 13, 32, 21]

Time complexity concept:
what is time complexity in simple english?
It doesnot depend on the n value 
but it depends on the number of operations performed by the algorithm as the input size increases.
it depends on highest power of n in the algorithm.

for(i=0;i<n;i++)
{
    //some code
}
time complexity: O(n) because the loop runs n times.
for(i=0;i<n;i++)
{
    for(j=0;j<n;j++)
    {
        //some code
    }
}
time complexity: O(n^2) because the outer loop runs n times and for each iteration of the outer loop,
 the inner loop also runs n times, resulting in n * n = n^2 iterations.
note
for for loop  time complexity is O(n) because the loop runs n times.
 for nested for loop time complexity is O(n^k) where k is the number of nested loops.
